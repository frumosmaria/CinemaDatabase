🎬 Cinema Database System
A relational database schema for managing cinema operations: movies, screenings, bookings, and payments.

📌 Key Features
Normalized Schema: 8 interconnected tables ensuring data integrity.

Complete Workflow: From movie categorization to ticket issuance and payment tracking.

Analytics Ready: Includes queries for revenue, occupancy, and customer behavior.

🏗️ Database Structure
Catalog: genres, movies

Infrastructure: rooms, seats

Operations: screenings

Sales: customers, tickets, payments

🚀 Quick Start
Ensure MySQL Server is running.

Run the provided .sql script in your favorite SQL client (Workbench, DBeaver).

The script will automatically create the cinema database and populate it with sample data.

📊 Sample Insights Included
The script features pre-built queries for:

Revenue: Total earnings per movie and global totals.

Popularity: Identifying top-selling films using GROUP BY and HAVING.

Reporting: Customer purchase history and seating charts.

🛠️ Tech Stack & Tools
Engine: MySQL Server

IDE: MySQL Workbench (Used for schema design and query execution)

Language: SQL
